# Gap: STORY-003a тАФ 2026-06-15
## What changed
The story spec describes `academic.json` as having a flat `gpa` field (e.g., `{ gpa: 3.9, confidence: 92, source: "transcript.pdf" }`). The actual file produced by STORY-003's extraction pipeline uses a rich nested schema: `academicData.gpa.overall.value`, `academicData.gpa.overall.confidence`, `academicData.gpa.overall.source`. The `GET /api/essays/provenance` endpoint and the `_resolveProvenanceUsed` helper in `POST /api/essays/generate` were updated to handle both schemas (rich nested and flat), with priority given to the nested schema.

## Files affected
- src/server/routes/essays.js тАФ dual-schema GPA handling in both the provenance GET and the _resolveProvenanceUsed helper

## Side-effects on other stories
None. The GPA read logic is isolated to the provenance endpoints introduced in this story.

## Recommended spec update
Update architecture.md `## Data file structure` example to show the full `academic.json` schema as produced by STORY-003's extraction pipeline, so future stories referencing GPA data know which field paths to use.
